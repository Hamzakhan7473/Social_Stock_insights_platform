# Contributing to Social Stock Insights Platform

Thank you for your interest in contributing! This document provides guidelines for contributing to the project.

## Research-Driven Development

This project follows a research-driven approach, integrating findings from academic research in AI and finance. When contributing:

1. **Cite Research**: If implementing features based on research papers, include citations
2. **Document Decisions**: Explain the rationale behind architectural and algorithmic choices
3. **Maintain Quality**: Follow the quality scoring and fundamental analysis principles established in the research

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/Social_Stock_insights_platform.git`
3. Set up the development environment (see [SETUP.md](SETUP.md))
4. Create a feature branch: `git checkout -b feature/your-feature-name`

## Code Style

- **Python**: Follow PEP 8, use type hints where possible
- **TypeScript/React**: Follow ESLint rules, use TypeScript strictly
- **Commits**: Write clear, descriptive commit messages

## Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Include integration tests for API endpoints

## Pull Request Process

1. Update documentation if needed
2. Add tests for new features
3. Ensure CI passes
4. Write a clear PR description explaining:
   - What changes were made
   - Why (research rationale if applicable)
   - How to test

## Research Contributions

If you're contributing research-based features:

- Include references to relevant papers
- Document the methodology
- Explain how it integrates with existing systems
- Update RESEARCH_INTEGRATION.md if applicable

## Questions?

Open an issue for questions or discussions about contributions.
